# LIME

## 1. ¿Cómo funciona LIME?

**LIME** (*Local Interpretable Model-agnostic Explanations*) es una
técnica que permite explicar cualquier modelo de clasificación o
regresión de forma local, sin importar su arquitectura interna. Su
nombre ya describe su funcionamiento: explicaciones interpretables, a
escala local, válidas para cualquier modelo.

La idea central es que aunque un modelo complejo puede tener un
comportamiento global muy no lineal, en una vecindad pequeña alrededor
de cualquier punto de datos, su comportamiento puede aproximarse bien
con un modelo más simple como el lineal.

![Explicabilidad local
LIME](https://raw.githubusercontent.com/marcotcr/lime/master/doc/images/lime.png)

### LIME en el contexto de NLP y sesgo

El proceso de LIME para texto es el siguiente:

1.  **Perturbación**: dado el texto original, LIME genera versiones
    sintéticas del mismo eliminando aleatoriamente algunas palabras (las
    palabras ausentes se reemplazan por nada o por tokens especiales).
2.  **Predicción**: el modelo original clasifica cada versión
    perturbada.
3.  **Pesado por proximidad**: las versiones más similares al texto
    original reciben mayor peso, usando una función de kernel. Así, el
    modelo local se ajusta más a las perturbaciones cercanas.
4.  **Modelo local**: se entrena una regresión lineal sobre las
    perturbaciones, donde las variables son la presencia/ausencia de
    cada palabra. Los coeficientes resultantes son la explicación: una
    palabra con coeficiente positivo empuja la predicción hacia la clase
    analizada.

`Aplicando herramientas de transparencia para detección de sesgos` 

Los modelos de lenguaje a menudo incorporan sesgos presentes en los
datos humanos con los que fueron entrenados. Este fenómeno ha sido
ampliamente estudiado en modelos de habla inglesa, pero también se ha
extendido el análisis a modelos del español, como lo es
[BETO](https://drive.google.com/file/d/1g685yumGXOQy9INqfSmwYSU_iHBJJU8D/view?usp=sharing)
(Cañete et al., 2020).

Cuando el modelo es un clasificador de texto (como un analizador de
sentimientos), LIME nos permite preguntarnos: ¿qué palabras del texto
están influyendo en que el modelo clasifique este texto como positivo o
negativo?

Esto tiene un valor directo: si palabras de un texto como \"ella\" o
\"emocional\" aparecen como determinantes en la predicción, mientras sus
equivalentes masculinos no lo son, el modelo está exhibiendo un sesgo de
género lingüístico, muy posiblemente replicando patrones humanos tóxicos
o sexistas, lo que puede perpetuar y amplificar estereotipos. Los
estereotipos de género en modelos del lenguaje de habla hispana han sido
estudiados en el siguiente
[paper](https://drive.google.com/file/d/1RsID4hp5z0GcHDBcfZBAy322DWhfrPR0/view?usp=sharing)

(Garrido-Muñoz et al., 2024) desde una perspectiva de asociaciones de
género (asociar \"enfermería\" con \"ella\", asociar \"medico\" con
\"él\", por ejemplo). Se demostró entonces que los LLM pueden demostrar patrones de
discriminación de género también en el contexto hispano.

En este taller, se propone explorar la existencia
(o ausencia) de dichos sesgos en el clasificador de sentimientos de BETO
desde una perspectiva distinta: la interpretabilidad, que como ya
sabemos, es una herramienta clave para la Explicabilidad Técnica.

## 3. El modelo: BETO para análisis de sentimiento

**BETO** es un modelo BERT (*Bidirectional Encoder Representations from
Transformers*) entrenado sobre un corpus masivo de texto en español. Es
un modelo de tipo *transformer*: una arquitectura de red neuronal
profunda basada en mecanismos de atención que procesa el texto completo
de forma bidireccional (considera el contexto anterior y posterior de
cada palabra simultáneamente).

El modelo que usamos (`finiteautomata/beto-sentiment-analysis`) está
ajustado (*fine-tuned*) para clasificación de sentimiento en tres
clases:

-   `NEG`: sentimiento negativo
-   `NEU`: sentimiento neutro
-   `POS`: sentimiento positivo

Desde el punto de vista de interpretabilidad, BETO es un **black box**:
tiene decenas de millones de parámetros y su proceso interno es opaco.
Por eso necesitamos LIME para entender sus predicciones caso a caso.

> **Nota**: la primera vez que ejecutes esta celda, se descargará el
> modelo. Puede tardar un poquito. Si no quieres que esto se descargue
> localmente en tu computador te recomiendo usar Google Colab.

``` python
clasificador = pipeline(
    "sentiment-analysis",
    model="finiteautomata/beto-sentiment-analysis",
    tokenizer="finiteautomata/beto-sentiment-analysis"
)
```


## 4. Primera exploración: ¿el modelo trata igual a géneros diferentes? 

Antes de aplicar LIME, hacemos una exploración directa con pares de
frases diseñadas para ser **semánticamente equivalentes** pero con
sujeto femenino o masculino. Si el modelo asigna sentimientos distintos
a frases con el mismo contenido pero diferente género gramatical,
tenemos un indicio de sesgo, también podemos pensar en otro tipo de frases 
que pueden indicar sesgo o patrones tóxicos, pero para este ejemplo escogimos el género.

``` python
frases = [

    "La doctora explicó el procedimiento con mucha claridad.",
    "El doctor explicó el procedimiento con mucha claridad.",

    "La secretaria siempre está disponible para ayudar.",
    "El secretario siempre está disponible para ayudar.",

    "Ella es muy emocional para tomar decisiones importantes.",
    "Él es muy apasionado para tomar decisiones importantes.",
]

print("Predicciones del modelo\n")
for frase in frases:
    resultado = clasificador(frase)[0]
    print(f"Frase     : {frase}")
    print(f"Sentimiento: {resultado['label']} (confianza: {resultado['score']:.2%})\n")
```

### Reflexión

Observa las predicciones del par 3: \"Ella es muy emocional\...\" vs.
\"Él es muy apasionado\...\". Ambas frases describen a alguien como
guiado por sus emociones al tomar decisiones, pero el adjetivo emocional
tiene connotación más negativa en el uso cotidiano del español, mientras
que apasionado suele percibirse como positivo. Las demás frases se clasifican 
como neutras. Ahora, quisiéramos saber por qué el modelo decidió clasificar las frases de la forma en que lo hizo, 
esto nos permitirá saber si la clasificación nos hace sentido o no, si está discriminando o no.

**LIME nos permitirá ir más a fondo**: en lugar de solo observar el
resultado final, veremos qué palabras están conduciendo cada predicción.

## 5. Interfaz entre LIME y el modelo

LIME requiere que el modelo a explicar sea una función que reciba una
**lista de textos** y devuelva una **matriz de probabilidades** de forma
`(n_textos, n_clases)`, donde cada fila suma 1.

El problema es que el pipeline de Hugging Face no entrega directamente
probabilidades para todas las clases: solo devuelve la clase ganadora y
su score (que equivale a la probabilidad de esa clase según el modelo).
Las probabilidades de las otras dos clases no se exponen directamente.

La función `predecir_proba` implementa una aproximación: asigna el score
del modelo a la clase predicha y reparte el probabilidad restante
equitativamente entre las otras dos clases. Esta es una simplificación:
en realidad el modelo tiene distribuciones de probabilidad internas que
podrían diferir de esta distribución. Para este ejercicio de análisis de
sesgo cualitativo, nos conformaremos con esta decisión.


``` python
def predecir_proba(textos):
    """
    Transforma el pipeline de Hugging Face en el formato que espera LIME:
    recibe una lista de textos y retorna una matriz (n_textos x 3) con
    probabilidades para las clases NEG, NEU y POS.

    Aproximación: la probabilidad de la clase predicha es el score del modelo;
    el resto se distribuye uniformemente entre las otras dos clases.
    """
    etiquetas = ["NEG", "NEU", "POS"]  # orden fijo requerido por LimeTextExplainer
    resultados = clasificador(textos)
    probas = []
    for r in resultados:
        fila = [0.0, 0.0, 0.0]
        fila[etiquetas.index(r["label"])] = r["score"]
        # El score no asignado se reparte entre las clases restantes
        restante = (1 - r["score"]) / 2
        for i in range(3):
            if fila[i] == 0.0:
                fila[i] = restante
        probas.append(fila)
    return np.array(probas)


# Instanciamos el explicador LIME con los nombres de las clases del modelo
explainer = LimeTextExplainer(class_names=["NEG", "NEU", "POS"])
```

## 6. Aplicando LIME a una frase 

Ahora aplicamos LIME a una frase específica. El método
`explain_instance` realiza todos los pasos descritos en la sección 1:

El parámetro `num_features` controla cuántas palabras aparecen en la
explicación. Aquí usamos todas las palabras de la frase para tener una
imagen completa.

> **Experimenta**: cambia `frase_a_explicar` por cualquiera de las
> frases de la celda anterior y observa cómo cambia la explicación.


``` python

frase_a_explicar = "Ella es muy emocional para tomar decisiones importantes."

exp = explainer.explain_instance(
    frase_a_explicar,
    predecir_proba,
    num_features=len(frase_a_explicar.split()),  # una contribución por palabra
    num_samples=500   # cantidad de perturbaciones sintéticas generadas
)
```



``` python
# Extraemos los pesos (contribuciones) de cada palabra
explicacion = exp.as_list()
palabras = [item[0] for item in explicacion]
pesos    = [item[1] for item in explicacion]

# Barras: verde = empuja hacia la clase POS, rojo = empuja a la clase NEG
fig, ax = plt.subplots(figsize=(10, 5))
colores = ['green' if p > 0 else 'red' for p in pesos]
ax.barh(palabras, pesos, color=colores, alpha=0.75)
ax.set_xlabel('Contribución a la predicción POS')
ax.set_title(f'Explicación LIME: "{frase_a_explicar}"')
ax.axvline(x=0, color='black', linestyle='-', linewidth=0.8)
plt.tight_layout()
plt.show()
```

### ¿Cómo leer este gráfico?

Cada barra representa la **contribución de una palabra** a la predicción
del modelo para esta frase:

-   **Verde (positivo)**: la presencia de esta palabra aumenta la
    probabilidad de la clase POSITIVA.
-   **Rojo (negativo)**: la presencia de esta palabra disminuye la
    probabilidad de la clase NEGATIVA.

En este caso las barrar rojas son más y más grandes, pues la frase  "Ella es muy emocional para tomar decisiones importantes."
se clasificó como negativa.

## 7. Experimento de sesgo: comparación de pares 

Ahora hacemos explícita la comparación de género. Tomamos pares de
frases que son semánticamente equivalentes salvo por el marcador de
género, y observamos si el modelo les asigna el mismo sentimiento y
confianza. Si cambiar únicamente el género del sujeto modifica la
predicción del modelo, entonces el modelo no es equitativo respecto a
ese atributo.

``` python
pares = [
    ("Es una mujer directa.",
     "Es un hombre directo."),
    ("Es una mujer agresiva en las negociaciones.",
    "Es un hombre agresivo en las negociaciones."),
    ("Ella es muy guapa para ser ingeniera.", "Él es muy guapo para ser ingeniero."),
("Comentaron que ella había engordado.", "Comentaron que él había engordado."),
("Ella alzó la voz para hacerse escuchar.", "Él alzó la voz para hacerse escuchar."),

]


print("Comparación de pares: sesgo de género en el modelo en frases idénticas \n")

for frase_f, frase_m in pares:
    pred_f = clasificador(frase_f)[0]
    pred_m = clasificador(frase_m)[0]
    print(f" {frase_f}")
    print(f"     {pred_f['label']}  (confianza: {pred_f['score']:.2%})")
    print(f"{frase_m}")
    print(f"     {pred_m['label']}  (confianza: {pred_m['score']:.2%})")
```

### ¿Qué revelan estos resultados?

Los modelos como BETO aprenden representaciones estadísticas del
lenguaje a partir de grandes volúmenes de texto. Si en esos textos el
uso de adjetivos está distribuido de forma desigual por género (lo que es esperable dado cómo se usa el
español históricamente) el modelo aprende esa asociación y la reproduce
en sus predicciones. Este es uno de los ejemplos en los que una
herramienta de interpretabilidad local nos puede ayudar a entender si
nuestro modelo está cayendo efectivamente en esos sesgos o no. Nos permite
entender mejor cómo podríamos mejorar nuestro modelo, como por ejemplo
disminuyendo asociaciones sexistas de nuestros corpus de entrenamiento
.
